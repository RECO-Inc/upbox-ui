# Color

디자인 시스템의 색상 토큰은 **3-layer 구조**로 정의됩니다.

1. **Primitive** — Figma 팔레트 원본 (`grey-*`, `navy-*`, `blue-*`, `green-*`, `red-*`, `orange-*`). 각 컬러 계열당 20-100 스케일.
2. **Semantic** — 의미 기반 매핑. `cta/primary`, `status/negative`, `label/1` 등. Primitive를 참조하되 의미에 맞게 이름이 부여됩니다.
3. **Component utility** — 실제 컴포넌트에서 사용되는 Tailwind 클래스 (`bg-navy-80`, `text-grey-60`). Tailwind preset을 통해 Primitive/Semantic 토큰과 자동 연결됩니다.

## 사용 규칙

- **컴포넌트 내부**: 가능한 한 Semantic 토큰 먼저 (`bg-cta-primary` vs `bg-navy-80`). 없으면 Primitive 사용.
- **네이밍은 Figma 기준**: 과거 `base/primary/info/positive/error/warning` → 현재 `grey/navy/blue/green/red/orange`로 통일 완료.
- **하드코딩 금지**: `#E11D1D` 직접 기입은 피하고 `text-red-80` 또는 `text-status-negative` 사용.
- **Dark mode**: `tokens.css`의 `.dark` 스코프에서 grey 계열이 반전됩니다. 실제 컴포넌트는 토큰 이름 그대로 쓰면 자동 전환.

## Do / Don't

✅ **Do**
```html
<button class="bg-navy-80 text-grey-10 hover:bg-navy-90">확인</button>
<p class="text-red-80">오류 메시지</p>
```

❌ **Don't**
```html
<!-- hex 하드코딩 -->
<button class="bg-[#264870]">확인</button>

<!-- 과거 네이밍 -->
<button class="bg-primary-80">확인</button>
```
