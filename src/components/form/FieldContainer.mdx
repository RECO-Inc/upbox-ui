# FieldContainer

**폼 필드의 표준 진입점.** label / control / description / error를 한 번에 조립하고, vee-validate Field와 자동 연결합니다.

> Figma의 `InputContainer` 정의를 코드로 구현한 래퍼. Input / Textarea / Select 등을 내부 slot으로 받습니다.

## 왜 필요한가

기존에는 아래처럼 5~6개 primitive를 매 필드마다 직접 조합해야 했습니다:

```vue
<FormField name="email">
  <FormItem>
    <FormLabel>Email</FormLabel>
    <FormControl>
      <Input type="email" />
    </FormControl>
    <FormDescription>설명</FormDescription>
    <FormMessage />
  </FormItem>
</FormField>
```

`<FieldContainer>`는 이를 한 줄로 축약합니다:

```vue
<FieldContainer name="email" label="Email" description="설명" required>
  <Input type="email" />
</FieldContainer>
```

## 자동 바인딩 (Option A 패턴)

- FieldContainer 내부는 vee-validate `<Field>`를 감싸고, `FIELD_CONTROL_INJECTION_KEY`로 바인딩을 provide
- Input / Textarea / Select는 해당 컨텍스트를 inject해서 **자동으로 v-model 연결** — `v-bind="componentField"` 보일러플레이트 불필요
- FieldContainer 밖에서는 inject가 null이므로 local v-model로 동작

## 핵심 동작

### 1. Description ↔ Error 상호 배제

error가 발생하면 description은 자동으로 숨김. 규칙을 선언적으로 적용.

```vue
<FieldContainer
  name="email"
  label="Email"
  description="도움말"
  :rules="required"
>
  <Input type="email" />
</FieldContainer>
```

### 2. Size 전파

`size` prop이 내부 FormLabel/FormDescription/FormMessage/Input/Textarea에 전파되어 Figma 타이포·패딩 스펙 유지.

```vue
<FieldContainer size="small" name="zipcode" label="우편번호">
  <Input size="small" />
</FieldContainer>
```

### 3. Required 마크

`required` prop → label 앞에 `*` (red-80). 시각 전용이며 실제 검증은 `rules`로 별도 지정.

## Slots

- `default` — 컨트롤 (Input/Textarea/Select 등). 스코프드 슬롯으로 `componentField`, `field`, `value`, `meta`, `errorMessage` 노출
- `tooltip` — label 옆 툴팁 아이콘 자리

## 사용 레벨

**Level 1 (대부분의 케이스)** — slot 자동 바인딩:
```vue
<FieldContainer name="email" label="Email">
  <Input type="email" />
</FieldContainer>
```

**Level 2** — 커스텀 컨트롤 / 네이티브 input:
```vue
<FieldContainer name="plain" label="Plain" v-slot="{ field }">
  <input v-bind="field" class="..." />
</FieldContainer>
```

**Level 3** — compositional primitives 직접 사용 (거의 안 씀):
```vue
<FormField name="email"><FormItem>...</FormItem></FormField>
```

## Do / Don't

✅ **Do**
```vue
<Form :validation-schema="schema" @submit="onSubmit">
  <FieldContainer name="email" label="Email" required>
    <Input type="email" />
  </FieldContainer>
</Form>
```

❌ **Don't** — FieldContainer를 deprecated Input props와 혼용
```vue
<!-- label은 FieldContainer의 책임. Input에 label은 제거됨 -->
<FieldContainer name="email">
  <Input label="Email" />
</FieldContainer>
```

## 접근성

- `<FormLabel :for="formItemId">` ↔ `<input id="formItemId">` 자동 연결
- `aria-describedby` 는 description / error 상태에 맞게 동적 변경
- `aria-invalid` 는 error 존재 여부에 바인딩
