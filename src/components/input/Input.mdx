# Input

단행 텍스트 입력. **Figma `Text Field` 스펙을 그대로 반영**합니다.

## 언제 사용하나

- 단일 문자열/숫자/비밀번호 입력이 필요할 때
- 여러 줄 입력은 [Textarea](?path=/docs/components-textarea--docs) 참고
- **폼 컨텍스트 안에서는 반드시 [FieldContainer](?path=/docs/components-fieldcontainer--docs)와 함께** — label/description/error가 자동 연결됩니다

## Variant & Size

- `default` / `filled` / `bottomline` 3종
- `small (32)` / `regular (40)` / `large (48)` 3종

Figma 스펙: small `p-8 text-12`, regular `px-16 text-14`, large `px-16 text-16`.

## 기능 props

| Prop | 설명 |
|---|---|
| `clearable` | 입력 있을 때 우측에 X 버튼 — 한 번에 비우기 |
| `password` | 마스킹 + 우측 눈 아이콘으로 visibility toggle |
| `subfix` | 우측에 단위/접미사 (예: `원`, `%`) |
| `counter` + `maxLength` | 입력값 길이 표시. `byteMode`로 byte 단위 전환 |
| `error` | 빨간 border (메시지는 FieldContainer가 담당) |

## 상태

| 상태 | 배경 | Border | 텍스트 |
|---|---|---|---|
| Default | grey-10 | grey-40 | grey-80 |
| Hover | - | grey-50 | - |
| Focus | - | blue-80 (ring blue-40) | - |
| Error | - | red-80 | - |
| Disabled | grey-20 | grey-40 | **grey-50 (dim)** |
| Readonly | grey-20 | grey-40 | grey-80 |

Disabled와 Readonly는 **텍스트 색상만** 다릅니다. 배경/테두리 동일.

## Do / Don't

✅ **Do** — FieldContainer로 감싸기
```vue
<FieldContainer name="email" label="Email" required>
  <Input type="email" placeholder="you@example.com" />
</FieldContainer>
```

✅ **Do** — standalone (레이블 불필요 시)
```vue
<Input v-model="query" placeholder="검색..." aria-label="검색어" />
```

❌ **Don't** — 과거 deprecated props 사용 금지
```vue
<!-- label/required/error-message 는 제거됨 -->
<Input label="Email" required error-message="..." />
```

## 접근성

- `inheritAttrs: false` + `v-bind="$attrs"` on inner input → FieldContainer가 전달하는 `id`, `aria-describedby`, `aria-invalid`가 실제 `<input>` 엘리먼트에 도달
- `aria-label` 직접 전달 가능 (레이블 없는 경우)
- 키보드 포커스 시 `focus-visible:ring-2 ring-blue-40`
