# Button

사용자의 행동을 유도하는 핵심 프리미티브. **4개의 variant + 2개의 buttonStyle + 5개의 size**로 조합.

## Variant

| Variant | 용도 |
|---|---|
| `primary` | 핵심 CTA — 한 화면에 1~2개 권장 |
| `destructive` | 되돌릴 수 없는 파괴적 행동 (삭제, 영구 종료) |
| `secondary` | 보조 CTA, 주 행동과 균형 필요할 때 |
| `tertiary` | 최소 emphasis, 인라인 보조 또는 ghost 필요시 |

## Button Style

- `filled` (기본) — 컬러 배경 채움
- `outlined` — 테두리만, 배경 투명 (약한 CTA, 카드 내 다중 액션)

## Size

`xsmall (24)` / `small (32)` / `regular (40, 기본)` / `large (48)` / `xlarge (56)`

## Do / Don't

✅ **Do**
```vue
<Button variant="primary">저장</Button>
<Button variant="destructive">삭제</Button>
<Button variant="tertiary" button-style="outlined">취소</Button>
```

❌ **Don't**
```vue
<!-- 과거 variant 이름. 이제 유효하지 않음 -->
<Button variant="assistant">...</Button>
<Button variant="usually">...</Button>

<!-- 한 화면에 primary 여러 개 -->
<Button variant="primary">저장</Button>
<Button variant="primary">취소</Button>
```

## 접근성

- `type="button"` 기본값으로 의도치 않은 form submit 방지. form 내부에서 submit 용도라면 명시적 `type="submit"`
- disabled 상태는 `disabled` prop 사용. aria-disabled 자동 처리
- 키보드: Enter / Space 모두 활성화
- loading 상태: `:loading="true"` — 스피너 + 클릭 차단
